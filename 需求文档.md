# Chrome书签整理网站需求文档（更新版）

更新时间：2026-03-25

## 1. 产品目标

Chrome书签整理网站用于帮助用户对大量书签进行导入、分类、筛选、检测、去重和导出。
产品重点是：
- 降低书签管理成本
- 提升链接可用性维护效率
- 支持可视化拖拽整理与层级分类管理

## 2. 角色与范围

### 2.1 用户角色
- 普通用户（无需注册，基于浏览器本地存储使用）

### 2.2 数据范围
- 书签数据
- 分类数据（支持父子层级）
- 检测结果与检测配置（并发、超时、重试）

## 3. 核心功能

### 3.1 书签导入与基础管理
- 支持导入 Chrome 书签 HTML 文件
- 支持手动新增书签
- 支持编辑、删除书签
- 支持按关键词实时搜索书签
- 支持导出书签为 HTML

### 3.2 分类管理（树形）
- 支持新增/编辑/删除分类
- 支持父子分类（`parentId`）
- 左侧分类列表树形展示
- 支持展开/收起子分类
- 点击父分类时，右侧展示该分类及全部子分类书签
- 分类统计按“自身+子分类”汇总

### 3.3 拖拽交互
- 书签卡片支持拖拽排序（同列表可重排）
- 书签卡片可拖拽到左侧分类，实现快速改分类
- 左侧分类支持同级拖拽排序（基于 `sortOrder` 持久化）

### 3.4 有效性检测
- 支持首次检测 / 再次检测 / 失效重检 / 停止检测
- 支持按并发、超时、重试次数检测
- 支持状态筛选（全部、有效、失效、未知、重复、未检测）
- 支持删除失效链接：
  - 主书签库来源：删除主库记录
  - 检测工作台来源：仅删除工作台记录

### 3.5 重复检测
- 重复判定规则：按“网站全称 URL 一致”判定重复（忽略 hash）
- 重复项支持列表查看与批量删除

### 3.6 撤销与数据安全
- 支持撤销：新增、删除、更新、导入、删除分类、清空
- 清空操作：
  - 同步清空书签与分类（重置为默认分类“未分类”）
  - 支持撤销恢复（同时恢复书签与分类）

### 3.7 主题与视觉
- 支持基础外观设置（主题切换、字体大小）
- 导航栏固定在顶部
- 背景与整体视觉支持“科技感”风格

## 4. 数据结构要求

### 4.1 书签结构
```js
const bookmark = {
  id: string,
  title: string,
  url: string,
  category: string,
  favicon: string,
  originalPath: string,
  createdAt: string,
  updatedAt: string
}
```

### 4.2 分类结构
```js
const category = {
  id: string,
  name: string,
  color: string,
  parentId: string | null,
  sortOrder: number,
  createdAt: string,
  updatedAt?: string
}
```

### 4.3 检测行结构（核心字段）
```js
const checkRow = {
  id: string,
  url: string,
  finalStatus: 'valid' | 'invalid' | 'unknown' | 'unchecked',
  sourceType: 'library' | 'workspace',
  sourceBookmarkId: string | null,
  duplicateKey: string // 规范化后的完整 URL（去 hash）
}
```

## 5. 关键业务规则

- 导入时可识别重复并支持覆盖策略
- “再次检测”覆盖 `失效/未检测/未知`
- “失效重检”仅覆盖当前 `失效`
- `Failed to fetch` 等浏览器限制场景优先标记为未知，避免误判失效
- `opaque` 响应按配置策略处理（当前为可达优先）

## 6. 非功能要求

- 前端单页应用，浏览器本地运行
- 本地存储持久化（localStorage）
- 大量书签场景下，检测过程需保证 UI 可用性
  - 已采用节流刷新与事件委托优化

## 7. 技术栈

- HTML5 / CSS3 / JavaScript (ES6+)
- Tailwind CSS (CDN)
- Font Awesome
- localStorage
- FileReader API
- HTML5 Drag and Drop API
